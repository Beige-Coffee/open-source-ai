An extension of `LoRA` that stores the frozen base model in 4-bit
precision using NF4 (a normal-distribution-tuned <G term="quantization">quantization</G> scheme) and
double-quantizes the <G term="quantization">quantization</G> constants themselves to save more
memory. <G term="lora">LoRA</G> adapters in full precision sit alongside the quantized base
and absorb the gradient.

The combined memory footprint is small enough that 65B parameter base
models fit on a single 48GB <G term="gpu">GPU</G> and 7B models fit on a 24GB consumer
card. The original paper showed QLoRA-tuned models matching full
<G term="fine-tuning">fine-tuning</G> quality on the Vicuna <G term="benchmark">benchmark</G>.

In practice QLoRA is the default for hobbyist <G term="fine-tuning">fine-tuning</G>. <G term="unsloth">Unsloth</G>,
<G term="axolotl">Axolotl</G>, and <G term="trl">TRL</G> all expose it as a configuration flag. The trade-off
versus <G term="lora">LoRA</G> is slower training (the quantize-dequantize step adds
overhead) in exchange for fitting bigger bases on smaller hardware.
