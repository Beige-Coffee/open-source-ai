## What this layer is

The orchestration layer that turns a model into an actor. An agent is a
program that uses an LLM to make decisions about which tools to call, in
what sequence, with what arguments, until some goal state is reached.
The agent layer is where most of the user-visible AI experience lives
in 2026: Claude Code, Cursor, Devin, Goose, Aider, Continue, Cline,
OpenHands are all agents, and the difference between "model" and "agent"
is largely the difference between "asks one question" and "does a thing."

This is the most contested layer for sovereignty because it is the
layer closest to the user. Whoever owns the agent owns the user
relationship, the workflow integration, the data trail, and the ability
to swap the underlying model. The model behind your IDE could change
tomorrow and you might never notice; the agent in front of it is what
you see.

Functionally the layer splits into two clusters. **Agent frameworks**
(LangChain, LlamaIndex, AutoGen, CrewAI) are libraries for building
agents. **Agent products** (Goose, OpenHands, Aider, Cline, Continue,
Claude Code, Cursor, Devin) are end-user agents that solve specific
workflows (mostly coding in 2026).

## Open vs. closed

The open agent landscape in 2026 has a few strong projects, mostly
coding-focused. **Goose** (originally from Block, recently transferred
to the Linux Foundation's Agentic AI Foundation) is the most explicitly
sovereignty-positioned open agent: extension-based, runs anywhere, BYOK
on the model, MCP-native. **OpenHands** (formerly OpenDevin) is an open
coding agent with a sandboxed execution environment that competes
head-to-head with Devin. **Aider** is the terminal-native open coding
agent, written by Paul Gauthier; the simplest production-quality open
agent and one of the most loved. **Cline** is a VS Code-native open
agent. **Continue** is an open IDE coding agent.

The closed side is dominated by frontier-lab agents and venture-backed
startups. **Claude Code** is Anthropic's first-party coding agent.
**Cursor** (the IDE) is closed-source and has integrated agentic
features. **Devin** (Cognition Labs) is the closed autonomous-software-engineer
pitch. Replit Agent, v0 (Vercel), and Lovable are all closed. The
frontier user-facing AI experience in 2026 is mostly proprietary.

The structural difference: open agents are typically thin wrappers
around BYOK model access. Closed agents own the full stack (subscription
plus workflow plus model). The closed agent has more leverage to
capture data, lock in workflows, and selectively expose model
capabilities. The open agent is interchangeable with the model behind
it.

## The lock-in vector

Closed agents defeat open weights. Even if you have open silicon, open
runtime, and open weights, if every user touches AI through a closed
agent (Claude Code, Cursor, Devin), the closed-agent vendor owns the
workflow and can choose what models they expose. The user is "using AI"
but is locked into one vendor's agent. The model substitutability that
open weights promised dies at the agent layer.

There are three structural lock-ins at this layer. **Model and decision
lock-in**: the vendor picks the model and can swap it tomorrow. **Data
surveillance**: closed agents see everything (prompts, code, file
contents, conversation history). **Accumulated context**: closed agents
build memory about you (Cursor remembers your codebase, ChatGPT
remembers your style) that is typically not exportable. Even if a
better open agent ships tomorrow, switching means starting over. Sunk-
context cost is the deepest reason people do not switch.

The framing here is the sharpest at any layer: open weights without open
agents is a museum exhibit, because end users do not interact with
weights, they interact with the agent in front of the weights.
